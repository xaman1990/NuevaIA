# MVP Lock — Agent OS

## Objetivo
Evitar que el MVP vuelva a inflarse por entusiasmo técnico o presión comercial difusa.

## Alcance aprobado
### Incluido
1. Workspace y membresía básica
2. Registro de agente
3. Versionado de agente
4. Catálogo de tools
5. Asignación de tools por versión de agente
6. Endpoint para crear run
7. Ejecución asíncrona simple
8. Persistencia de estado de run
9. Persistencia de pasos de traza
10. Consulta y exportación de auditoría
11. UI mínima para agentes y runs

### Excluido
1. Marketplace
2. Billing usage-based
3. Memoria semántica avanzada
4. Coordinación multiagente
5. Sistema de plugins abierto
6. Evaluación automática sofisticada
7. Approval workflows complejos
8. Kubernetes en primera iteración
9. Multicloud
10. SDKs externos completos

## Regla de cambio
Ningún elemento excluido entra al MVP sin revisión conjunta de CTO, CPO y CEO.

## Regla de prioridad
Si una tarea no mejora control, trazabilidad, seguridad u operabilidad, no entra.
