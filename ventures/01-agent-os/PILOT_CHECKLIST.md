# Pilot Checklist

## Before pilot
- customer has real agents or active pilot work
- one risky workflow is selected
- buyer and technical owner are known
- success metrics are agreed

## During pilot
- agent registry is working
- run trace is visible
- tool access is limited
- audit output is usable
- onboarding time is tracked

## After pilot
- customer can name clear value
- at least one incident or risk was easier to diagnose
- next expansion path is identified
- annual follow-up discussion is open

## Red flags
- pilot is treated as a demo only
- no real workflow is connected
- no owner on customer side
- too much custom work is required
